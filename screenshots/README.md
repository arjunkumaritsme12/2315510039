# Screenshots Checklist

This folder contains the submission-ready screenshot checklist and references for the full-stack notification system.

## Required Screenshots
1. API Response: capture the notification payload returned by the evaluation API.
2. Insomnia Testing: show a successful GET request with headers.
3. Terminal Output: capture the Stage 6 priority script console table.
4. All Notifications Page: capture the responsive list with filters and pagination.
5. Priority Inbox: capture the top 10 ranked cards with scores.
6. Filtering: capture search and type-filter behavior.
7. Pagination: capture the page navigation controls.
8. Viewed vs Unviewed: capture the green and gray indicators.
9. Mobile View: capture the drawer and stacked layout.
10. Desktop View: capture the full-grid experience.
11. Loading State: capture the circular loader.
12. Error State: capture the snackbar or warning banner when the API is unavailable.

## Recommended Capture Commands
- Frontend: `npm run dev -- --host 127.0.0.1`
- Stage 6 script: `node priority_notifications.js`
